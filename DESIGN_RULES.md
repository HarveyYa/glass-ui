# Glass UI 设计约束（DESIGN_RULES）

本主题的所有改动都必须遵守以下约束。新增/修改样式前先对照本文件。

## 硬规则

1. **只改 CSS**
   主题仅由 CSS 覆盖实现 —— 不写 JS、不封装组件、不改 DOM 结构。
   只有当某效果用 CSS 确实无法实现时，才允许其他手段，且必须在本文件「例外」一节登记。

2. **玻璃风格**
   半透明 + `backdrop-filter` 模糊 是核心的「表面处理」。任何「面」都应是透明玻璃，不用不透明实色背景（明确例外见下）。

3. **遵循 Element Plus**
   不无中生有，也不随意剥除。
   - EP 某处有的处理（渐变 / 高光 / 层次 / affordance）→ 我们**保留并玻璃化**。
   - EP 没有的样式 → 我们**也不加**。
   - 沿用 EP 的组件结构、交互反馈位置、以及 `--el-*` 变量体系；能用 EP 自带变量就别硬覆盖组件内部。

## 冲突优先级（规则 2 与 3 打架时）

当「玻璃」与「跟随 EP」冲突（如 EP 卡片自带不透明白底 + 投影 + 圆角）：

| 方面 | 谁优先 |
|------|--------|
| 背景透明度、模糊、是否用重投影 | **玻璃规则优先** |
| 结构 / 反馈位置 / 颜色出现的位置 / 尺寸 / 间距 | **EP 优先** |

> 一句话：**把 EP「玻璃化」，而不是重新设计 EP。**

## 颜色（跟随 EP，不再强制单色）

- **强调色**：一个可配置的强调色（默认青 `#3DB4E5`），用在 EP 原本放「主色」的位置（激活 / 选中 / 聚焦 / 进度 / 勾选）。
- **其余颜色跟随 EP 的位置**：EP 在哪用色我们就在哪用色，只是纳入本主题的取值（如语义色采用去饱和版本）。中性的「面」保持透明玻璃，不强加新颜色。
- **渐变**：跟随 EP —— EP 有渐变则保留；若是中性组件的彩色渐变，换成**同色的透明度渐变**（如白 8%→2% 的高光过渡），保留「有层次」但不引入新颜色。

## 设计令牌（档位变量）

所有档位都以**命名 CSS 变量**定义在 `:root`。**禁止在规则里直接写魔法数字** —— 一律引用这些令牌。实现时这些令牌再喂给 EP 的 `--el-*` 变量。

### 强调色（公开旋钮）

| 变量 | 默认值 | 用途 |
|------|--------|------|
| `--glass-accent` | `#3DB4E5` | 唯一强调色，喂给 EP 主色位置（激活/选中/聚焦/进度/勾选） |

### 玻璃浓度 — 背景透明度档位

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-bg-surface` | `rgba(22,24,28, .22)` | 卡片 / 面板 |
| `--glass-bg-overlay` | `rgba(26,28,32, .55)` | 弹窗 / 抽屉 / 浮层外壳 |

### 填充档位（hover / 选中底，范围 .05–.12）

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-fill-1` | `rgba(255,255,255, .05)` | 输入框/默认按钮底、弱 hover |
| `--glass-fill-2` | `rgba(255,255,255, .08)` | 中 hover |
| `--glass-fill-3` | `rgba(255,255,255, .12)` | 强 hover / 按下 |

### 模糊档位（公开旋钮）

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-blur-sm` | `6px` | 输入 / 按钮 |
| `--glass-blur-md` | `20px` | 卡片 |
| `--glass-blur-lg` | `22px` | 浮层 / 弹窗 |

### 边框档位

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-border-1` | `rgba(255,255,255, .08)` | 细分隔线 |
| `--glass-border-2` | `rgba(255,255,255, .14)` | 常规边框 |
| `--glass-border-3` | `rgba(255,255,255, .30)` | 聚焦 / hover 边框 |

### 文字明度阶

近纯白冷调，按明度分级（喂给 EP 的 `--el-text-color-*`）。

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-text-1` | `#FEFFFF` | 主文字 |
| `--glass-text-2` | `rgba(255,255,255, .82)` | 常规文字 |
| `--glass-text-3` | `rgba(255,255,255, .55)` | 次要 / 占位 |
| `--glass-text-4` | `rgba(255,255,255, .35)` | placeholder |
| `--glass-text-5` | `rgba(255,255,255, .25)` | 禁用 |

### 语义色（去饱和，跟随 EP 的状态位置）

仅用于表达状态，不做装饰；喂给 EP 的 `--el-color-success/warning/danger/info`。

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-success` | `#6f8a67` | 成功 |
| `--glass-warning` | `#a8925c` | 警告 |
| `--glass-danger` | `#a86a60` | 危险 / 错误 |
| `--glass-info` | `#868c94` | 信息 |
| `--glass-rate` | `#bfa86a` | 评分星（暖金） |

### 形状 / 过渡

| 变量 | 值 | 用途 |
|------|----|------|
| `--glass-radius` | `2px` | 统一圆角（接近直角，公开旋钮） |
| `--glass-transition` | `.18s` | 过渡时长（须落在 `.15–.2s` 内） |

### 配套约束

- **一个浮层只有一层玻璃**：外壳（`.el-popper`）用 `--glass-bg-overlay`，内层面板一律透明，禁止叠加变深。
- **投影**：尽量不用（玻璃靠模糊 + 细边表达层次）；EP 浮层必需的投影可保留极淡。
- **过渡**：只作用于 背景 / 边框 / 颜色，不做位移/缩放。

## 对外 API（可调旋钮）

公开、保证稳定的只有这几个令牌，用户覆盖即可定制：

- `--glass-accent` —— 强调色
- `--glass-blur-sm` / `--glass-blur-md` / `--glass-blur-lg` —— 模糊强度
- `--glass-bg-surface` / `--glass-bg-overlay` —— 玻璃浓度
- `--glass-radius` —— 圆角

其余令牌（`--glass-fill-*` / `--glass-border-*` / `--glass-transition` 等）为内部实现，不保证跨版本稳定。

## 边界与兼容

- **不写死 `body` 背景**（背景由使用者的页面提供）。
- **不设置字体**：主题不碰 `font-family`，沿用使用者 / EP 的字体（和「不写死背景」同理）。
- **不打包** 字体 / Vue / Element Plus（后两者为 peerDependencies）。
- 必须在**任意背景**下可用（深 / 浅 / 图片）。
- **可读性**：可读优先 + 文字阴影兜底，正文对比度不低于约 `3:1`。

## 实现约束（重构中补充）

这些是写 CSS 时必须遵守的工程约束：

- **颜色派生**：强调色 / 语义色只定义**一个基色令牌**（`--glass-accent` / `--glass-success` …），各级色阶（light-3/5/7/8/9、dark-2）一律用 `color-mix()` 从基色派生，**不手写色阶**。改一个基色，整套联动。
- **`saturate()` 用令牌**：玻璃的 `backdrop-filter` 统一为 `blur(<blur 令牌>) saturate(var(--glass-saturate))`（`--glass-saturate: 115%`），不随手写。
- **`-webkit-backdrop-filter` 必须成对**：每处 `backdrop-filter` 都要带 `-webkit-` 前缀（Safari）。
- **`!important` 克制**：只在 EP 特异性必须时使用（主要是浮层 `.el-popper` 及其内层）；其余一律不用。
- **不复位 `body` 布局**：主题只设 body 的文字色 + 可读性 `text-shadow`，**不设** margin / min-height / font-family（这些是使用者 / 复位层的事）。
- **浏览器基线**：依赖 `backdrop-filter` 与 `color-mix()`，目标 Chrome 111+ / Safari 16.2+ / Firefox 113+（约 2023+）。低于此：无 `backdrop-filter` 时退化为「半透明但不模糊」（仍可用）；无 `color-mix` 时强调色色阶失效（需评估是否提供回退）。
